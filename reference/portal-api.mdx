---
title: "Portal API Proxy"
description: "Reference guide for all AllCodex Portal Next.js API proxy routes."
---

The Portal acts as a secure Backend for Frontend (BFF) proxy. The browser never communicates directly with the database (AllCodex Core) or the AI brain (AllKnower). It sends requests to `/api/*` on the Portal, which parses authentication cookies server-side and forwards requests to the private backends.

---

## Lore CRUD

These routes proxy requests directly to the AllCodex Core ETAPI.

| Method | Path | Description |
| :--- | :--- | :--- |
| `GET` | `/api/lore` | Query lore notes. Supports full ETAPI search query syntax (e.g. `?q=#loreType=character`). |
| `POST` | `/api/lore` | Create a new note. Body includes `title`, `type`, `mime`, `loreType`, `templateId`, `parentNoteId`, and optional `attributes`. |
| `GET` | `/api/lore/[id]` | Retrieve note metadata, labels, and relations. |
| `PATCH` | `/api/lore/[id]` | Update core properties of a note (title, type, mime). |
| `DELETE` | `/api/lore/[id]` | Permanently delete a note. |

## Note Content, Revisions & Media

| Method | Path | Description |
| :--- | :--- | :--- |
| `GET` | `/api/lore/[id]/content` | Returns the raw HTML body of a note. |
| `PUT` | `/api/lore/[id]/content` | Updates the HTML body. **Requires `Content-Type: text/plain`**. |
| `GET` | `/api/lore/[id]/preview?mode=gm\|player` | Retrieves HTML content sanitized via `sanitizeLoreHtml()` (for GM view) or `sanitizePlayerView()` (which strips `.gm-only` CSS containers). |
| `GET` | `/api/lore/[id]/image` | Proxies image binary files from AllCodex Core. Cached 1 day. |
| `GET` | `/api/lore/[id]/backlinks` | Returns all notes linking to the current note (inbound relations or text links). |
| `GET` | `/api/lore/[id]/breadcrumbs` | Returns parent-child ancestor paths for navigation. |
| `GET` | `/api/lore/[id]/revisions` | Lists all revision metadata for a note. |
| `GET` | `/api/lore/[id]/revisions/[revId]/content` | Retrieves the body content of a specific past revision. |

## Maps & Geolocation

AllCodex supports uploading maps and plotting pins using geolocation coordinates stored on child notes.

| Method | Path | Description |
| :--- | :--- | :--- |
| `GET` | `/api/lore/[id]/map` | Builds map data (dimensions, image URL) and plots geolocation pins. |
| `POST` | `/api/lore/[id]/map/upload` | Uploads a map image binary for a note. |

## Search & Mentions

| Method | Path | Description |
| :--- | :--- | :--- |
| `GET` | `/api/lore/mention-search?q=<text>` | Autocomplete endpoint for `@`-mention linking inside the rich-text editor. |
| `GET` | `/api/lore/note-search?q=<text>&type=<type>` | Title search for dropdown pickers. |
| `POST` | `/api/lore/autolink` | Scans text for lore entity title matches and returns linking IDs. |
| `GET` | `/api/search?q=<query>&mode=etapi\|rag` | Run search queries. `etapi` mode runs keyword/tag search; `rag` mode runs vector search via AllKnower. |

## AI & Brain Dump

These routes proxy tasks to the **AllKnower** Elysia API.

| Method | Path | Description |
| :--- | :--- | :--- |
| `POST` | `/api/brain-dump` | Submits text for entity extraction. Mode can be `auto`, `review`, or `inbox`. |
| `POST` | `/api/brain-dump/stream` | Server-Sent Events (SSE) stream for progressive entity extraction. |
| `POST` | `/api/brain-dump/commit` | Commits approved entities from Review Mode. |
| `GET` | `/api/brain-dump/history` | Lists the 20 most recent brain dump jobs. |
| `GET` | `/api/brain-dump/history/[id]` | Retrieves full details of a specific brain dump. |
| `GET` | `/api/brain-dump/[id]/diffs` | Retrieves code-diff format changes generated by a brain dump. |
| `POST` | `/api/brain-dump/batch` | Queues bulk brain dump batch jobs. |
| `GET` | `/api/brain-dump/batch/[batchId]` | Status of a bulk background run. |
| `DELETE` | `/api/brain-dump/batch/[batchId]` | Cancels a running or queued bulk run. |

## AI Analysis & Copilot

| Method | Path | Description |
| :--- | :--- | :--- |
| `POST` | `/api/ai/consistency` | Runs a consistency check on selected notes. |
| `GET\|POST` | `/api/ai/gaps` | Scans a subtree to spot underdeveloped lore holes. |
| `POST` | `/api/ai/relationships` | Recommends new relationship links based on note text. |
| `PUT` | `/api/ai/relationships` | Applies approved relations to Core (creates bidirectional `~rel` attributes). |
| `POST` | `/api/lore/[id]/copilot/chat` | Submits chat messages to the contextual Article Copilot. |
| `POST` | `/api/lore/[id]/copilot/stream` | SSE stream for Article Copilot responses. |
| `POST` | `/api/lore/[id]/copilot/apply` | Applies the edit changes proposed by the Copilot. |

## Collections, Shares & System Configuration

| Method | Path | Description |
| :--- | :--- | :--- |
| `GET` | `/api/quests` | Lists notes tagged with `#quest`. |
| `GET` | `/api/statblocks` | Lists notes tagged with `#statblock`. |
| `GET` | `/api/timeline` | Lists notes categorized under event or timeline lore types. |
| `GET` | `/api/share/tree` | Lists all notes with draft, GM-only, and publication status. |
| `POST` | `/api/integrations/allcodex/connect` | Connects Portal to Core (saves encrypted token in AllKnower). |
| `POST` | `/api/config/wipe` | **DANGEROUS**: Drops vector search tables and clears tracking databases. |
| `GET` | `/api/usage/summary` | Lists token expenses by model and task. |
| `GET\|PUT` | `/api/usage/budgets` | Reads or writes daily warning and hard budget limits. |
